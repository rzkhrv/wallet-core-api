# TRON: возвращаем blockHeader, убираем blockId/blockNumber

## Контекст
- Требование: публичный API должен принимать полный `blockHeader` и не должен требовать `blockId`/`blockNumber`.
- Цель — убрать лишние параметры и использовать `blockHeader` как единственный источник TAPOS‑данных.

## Цель
Сделать build/sign для TRON так, чтобы:
- В DTO присутствовал обязательный `blockHeader`.
- Поля `blockId` и `blockNumber` были удалены из публичного API.
- Адаптер использовал `blockHeader` для заполнения `Transaction.blockHeader`.

## План работ
1) **DTO**
   - Добавить `TronBlockHeaderRequestDto` с полями `number`, `parentHash`, `txTrieRoot`, `witnessAddress`, `version`, `timestamp`.
   - В build DTO заменить `blockId`/`blockNumber` на обязательный `blockHeader`.
2) **Адаптер/сервис**
   - В adapter input заменить `blockId`/`blockNumber` на `blockHeader`.
   - Формировать `TW.Tron.Proto.BlockHeader` из полного `blockHeader`.
3) **Тесты**
   - Обновить unit‑тесты DTO/сервиса/адаптера/контроллера под новый контракт.
4) **Документация**
   - Обновить decision/troubleshooting, чтобы зафиксировать новый контракт.

## Риски
- Нужно убедиться, что клиент передаёт корректный `blockHeader` из той же сети/ноды, куда идёт broadcast.
