# TRON e2e: обновить тесты под blockHeader

## Контекст
- После перехода на обязательный `blockHeader` e2e‑тесты TRON падают с 400.
- Нужно обновить `test/app.e2e-spec.ts`, чтобы запросы соответствовали новому контракту.

## Цель
- В e2e‑тестах TRON добавить `blockHeader` во все build/sign запросы.
- Добиться успешного прохождения `npm run test:e2e`.

## План работ
1) Добавить общий фикстурный `blockHeader` в `test/app.e2e-spec.ts`.
2) Обновить TRON build‑transaction и build‑transfer e2e запросы.
3) Обновить TRON sign (trx/trc20) тесты, чтобы они использовали payload с `blockHeader`.
4) Прогнать `npm run test:e2e`.
